## 3. MAIN APP PAGE (/app)

### LAYOUT (Desktop)
Two column layout:

LEFT COLUMN (40% width):
- API Key section
- Provider selector
- Prompt input
- File upload
- Canvas size selector
- Generate button

RIGHT COLUMN (60% width):
- Canvas preview
- Playback controls
- Editor panel (appears after generation)
- Export panel (appears after generation)

### LAYOUT (Mobile)
Single column, stacked:
- All left column items first
- Canvas preview below
- Controls below canvas

---

### API KEY SECTION
Location: Top of left column

States:

State A - No API key:
- Yellow warning banner
- Text: "Add an API key to start generating"
- Button: "Add API Key" (yellow/warning color)

State B - API key saved:
- Green success indicator dot
- Text: "API Key: ••••••••1234" (masked)
- Small button: "Change" (gray, small)

"Add API Key" button behavior:
→ Opens API Key Modal (described below)

---

### API KEY MODAL
Triggered by: "Add API Key" button
Type: Centered modal with dark overlay

Elements:
- Title: "Configure AI Provider"
- Close button: X in top right corner

- Provider dropdown:
  Options: Gemini | OpenRouter | Groq | OpenAI | Anthropic
  Shows: Provider logo + name in each option
  Default: Gemini

- Model selector (appears after provider selected):
  Gemini selected → shows: "gemini-2.0-flash-exp (Free)"
  Groq selected → shows model dropdown of free models
  OpenRouter → shows free models list

- API Key input field:
  Placeholder: "Paste your API key here..."
  Type: password (dots shown, toggle to show)
  Eye icon: Toggle visibility

- Helper link:
  Text: "Get a free Gemini API key →"
  Behavior: Opens aistudio.google.com in new tab
  Changes based on selected provider

- Save button: "Save & Continue"
  Behavior: Saves to localStorage, closes modal, 
            enables prompt input

- Clear button: "Remove Key" (small, red text, no bg)
  Only shown when key already exists

---

### PROVIDER SELECTOR
Location: Below API key section (compact, horizontal)
Type: Small pill buttons in a row

Shows:
[Gemini] [OpenRouter] [Groq] [OpenAI] [Anthropic]

Active state: Indigo background, white text
Inactive state: Gray border, gray text
Behavior: Clicking switches active provider
Note: Only shown providers that have keys saved
If only one provider set up, this section is hidden

---

### PROMPT INPUT
Location: Main input area, left column

Elements:
- Label: "Describe your animation"
- Textarea:
  Placeholder: "e.g. Fitness app called FitPro, blue and 
               white, energetic and motivating..."
  Min height: 120px
  Max height: 200px (scrollable)
  Auto-resize as user types
  
- Character counter: "142 / 500" (bottom right of textarea)
  Color: Gray normally, Orange when over 400, Red at 500

- Prompt tips toggle:
  Small text below: "Need inspiration? See examples"
  Behavior: Expands a list of example prompts
  Example prompts shown:
  • "Food delivery app, red orange, fast and energetic"
  • "Meditation app called Calm, blue purple, peaceful"
  • "Fintech app PayFlow, dark blue, professional"
  • "Gaming app, neon green black, explosive"
  Clicking any example: Fills the textarea with that text

---

### FILE UPLOAD ZONE
Location: Below prompt input

Normal state:
- Dashed border box
- Icon: Upload cloud icon ↑
- Text: "Drop your logo here"
- Subtext: "PNG, SVG, JPG, JSON, Lottie • Max 5MB"
- Button inside: "Browse Files" (small, outlined)

Drag over state:
- Dashed border turns indigo
- Background turns very slightly indigo tinted
- Text: "Drop to upload"

File loaded state:
- Shows file thumbnail/preview on left
- Shows filename and size on right
- Remove button: X on top right of thumbnail
- Supported: PNG/JPG shows image preview
           SVG shows SVG preview
           JSON/Lottie shows small Lottie player preview

Error state:
- Red border
- "File too large" or "Unsupported format"
- Subtext: "Max 5MB. Supported: PNG, SVG, JPG, JSON, Lottie"

### GENERATE BUTTON
Location: Bottom of left column
Size: Full width

States:

State IDLE (ready):
- Background: Indigo gradient
- Icon: Sparkles ✨ on left
- Text: "Generate Animation"
- Hover: Slightly brighter, slight scale 1.01x

State GENERATING:
- Background: Gray (disabled)
- Icon: Spinning loader
- Text: Changes every few seconds:
  → "Reading your prompt..."
  → "Understanding your brand..."
  → "Building motion design..."
  → "Generating your animation..."
  → "Almost ready..."

State SUCCESS (briefly, then resets to IDLE):
- Background: Green for 1 second
- Icon: Checkmark ✓
- Text: "Animation Ready!"

State ERROR:
- Background: Red briefly
- Icon: X mark
- Text: "Generation Failed - Try Again"
- Resets to IDLE after 2 seconds

Disabled state (no API key):
- Background: Very dark gray
- Text: "Add API Key First"
- Cursor: not-allowed

---

### CANVAS PREVIEW
Location: Top of right column
Size: Fills available width maintaining aspect ratio

States:

State EMPTY:
- Dark gray background
- Centered content:
  - Icon: Play button (large, outlined)
  - Text: "Your animation will appear here"
  - Subtext: "Generate an animation to see the preview"

State GENERATING:
- Shows animated placeholder
- Pulsing indigo gradient background
- Loading bars animating
- Progress text: "Building your animation..."

State READY - PLAYING:
- Shows live animation on canvas
- No overlay
- Smooth 60fps render

State READY - PAUSED:
- Shows animation frozen at current frame
- Semi-transparent pause overlay in corner
  (subtle, doesn't block the view)

Canvas overlay elements (always visible when animation ready):
- Top right corner: Small badge showing canvas size "1080×1920"
- Bottom left corner: Duration badge "3.2s"

---

### PLAYBACK CONTROLS
Location: Directly below canvas
Type: Horizontal row of controls

Elements (left to right):

1. Restart button
   - Icon: Skip back ⏮
   - Behavior: Restart animation from beginning
   - Tooltip: "Restart"

2. Play/Pause button
   - Icon: ▶ Play or ⏸ Pause (toggles)
   - Size: Slightly larger than other buttons
   - Behavior: Toggle animation playback
   - Keyboard shortcut: Spacebar

3. Loop toggle button
   - Icon: Loop/repeat ↻
   - Active state: Indigo color (looping on)
   - Inactive state: Gray (plays once)
   - Default: Active (looping on)
   - Tooltip: "Loop animation"

4. Speed control
   - Label: "Speed"
   - Dropdown: 0.25x | 0.5x | 1x | 1.5x | 2x
   - Default: 1x
   - Behavior: Changes playback speed of preview

5. Progress bar
   - Shows current position in animation
   - Click anywhere on bar to seek
   - Shows current time / total time
   - Format: "1.2s / 3.5s"

---

### EDITOR PANEL
Location: Below playback controls, right column
Visible: Only after animation is generated
Type: Card with sections, collapsible

Header:
- Text: "Edit Animation"
- Icon: Sliders icon ⚙
- Collapse toggle: Chevron ∨ / ∧

The 5 editor controls (Pareto 20% → 80% value):

CONTROL 1: Speed
- Label: "Animation Speed"
- Type: Horizontal slider
- Min: 0.5x, Max: 2x, Step: 0.1x
- Default: Value from AI config
- Current value shown: "1.2x" next to slider
- Effect: Recalculates all timing, re-renders instantly
- No AI call needed

CONTROL 2: Energy Level
- Label: "Energy"
- Type: Horizontal slider  
- Left label: "Calm" Right label: "Explosive"
- Min: 1, Max: 10, Step: 1
- Shows: Current number (1-10)
- Effect: Changes easing curves, particle intensity, 
          re-renders instantly

CONTROL 3: Colors
- Label: "Color Palette"
- Shows: 4 color swatches in a row
  - Background color swatch
  - Primary color swatch
  - Secondary color swatch
  - Accent color swatch
- Clicking any swatch: Opens color picker
- Color picker type: Native browser color input + hex input
- Effect: Updates colors, re-renders instantly

CONTROL 4: Text
- Label: "Brand Text"
- Primary text input: Text field, shows current brand name
- Secondary text input: Text field, shows current tagline
- Character limit: 30 chars each
- Effect: Updates text, re-renders instantly
- Font weight toggle: Normal | Bold (two buttons)

CONTROL 5: Particles
- Label: "Particle Effects"
- Type: Toggle switch (on/off)
- Shows current particle count when on: "24 particles"
- Intensity slider (only shown when on):
  Min: Low, Max: High
- Effect: Toggles particles, re-renders instantly

REGENERATE SECTION:
Below all 5 controls:
- Divider line
- Text: "Not happy with the result?"
- Button: "Regenerate" (full width, outlined style)
  Icon: Refresh ↺
  Behavior: Calls AI again with same prompt,
            generates new variation
- Button: "Edit Prompt" (full width, ghost style)
  Behavior: Focuses the prompt textarea in left panel,
            scrolls up to it

---

### EXPORT PANEL
Location: Below editor panel
Visible: Only after animation is generated
Type: Card with 4 download buttons

Header:
- Text: "Download Animation"
- Icon: Download ↓

Four export buttons (each full width, stacked):

BUTTON 1: Lottie JSON
- Icon: {} code icon
- Label: "Lottie JSON"
- Sublabel: "Best for mobile & web apps • Vector • Tiny file"
- Badge: "RECOMMENDED" (green badge)
- Behavior: Generates Lottie JSON → downloads .json file
- Loading state: Spinner + "Preparing..." 
- File format: "animationname.json"

BUTTON 2: GIF
- Icon: GIF icon (or play circle)
- Label: "Animated GIF"
- Sublabel: "Works everywhere • Share anywhere"
- Behavior: Records canvas → encodes GIF → downloads
- Loading state: Progress bar + "Encoding... 45%"
- File format: "animationname.gif"

BUTTON 3: MP4 Video
- Icon: Video camera icon
- Label: "MP4 Video"
- Sublabel: "For social media & presentations"
- Behavior: Records canvas → downloads .mp4
- Loading state: Spinner + "Recording..."
- File format: "animationname.mp4"

BUTTON 4: CSS Animation
- Icon: CSS icon (or code brackets)
- Label: "CSS Animation"
- Sublabel: "Ready-to-use CSS for web projects"
- Behavior: Generates CSS keyframes → downloads .css
- Loading state: None (instant)
- File format: "animationname.css"

All buttons:
- Style: Dark gray background, white text
- Hover: Slightly lighter background
- Active/loading: Indigo background during processing
- Success: Brief green flash then back to normal

### HISTORY CARD
Each card contains:

TOP SECTION:
- Animation thumbnail (static image, first frame)
- On hover: Play button overlay appears over thumbnail
  Clicking play: Small preview plays in the card
- Aspect ratio: Matches original canvas size

BOTTOM SECTION:
- Prompt text: First 60 characters of original prompt
  Full text on hover (tooltip)
- Date: "2 hours ago" or "Jan 15, 2025"
- Duration badge: "3.2s" small pill
- Size badge: "Portrait" small pill

ACTION BUTTONS (appear on hover or always on mobile):
- "Load & Edit" button (primary, full width)
  Behavior: Navigate to /app with this animation loaded
- "Download" dropdown button (secondary)
  Dropdown shows: Lottie | GIF | MP4 | CSS
  Downloads that format immediately
- Delete button: Small trash icon (red, top-right corner)
  Behavior: Removes from localStorage, card disappears
  No confirmation needed (can always regenerate)

## 7. TOAST NOTIFICATIONS

System for feedback messages:
Location: Bottom right corner
Auto-dismiss: 4 seconds
Max shown: 3 at once

Types:

SUCCESS:
- Green left border
- Checkmark icon
- Examples: "Animation generated!", "File downloaded!"

ERROR:
- Red left border  
- X icon
- Examples: "API key invalid", "Generation failed"

WARNING:
- Yellow left border
- Warning triangle icon
- Examples: "Using fallback animation config"

INFO:
- Blue left border
- Info icon
- Examples: "Tip: Try adding your brand colors"

## 8. KEYBOARD SHORTCUTS

Space: Play/Pause animation
R: Restart animation
L: Toggle loop
1/2/3: Switch canvas size
G: Generate (when prompt filled)
Escape: Close any open modal
